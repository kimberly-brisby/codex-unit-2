# Level 2 — Headings & Outline

## Objective

Teach the semantic role of `h1`–`h6` and how to build a clear document outline.

## Benefits (SEO & accessibility)

- Clear, hierarchical headings help search engines understand page structure and surface relevant sections in search snippets.
- Proper heading order improves navigation for screen reader users and creates a predictable document outline.

## Complete these tasks

- **Ensure** a single top-level `h1` for the page.
- **Use** `h2`–`h6` for subsections in descending order (don't skip levels arbitrarily).

### Quick example

```html
<h1>Page title</h1>
<h2>Section A</h2>
<h3>Subsection A.1</h3>
```

## Hints (Socratic)

- How do heading levels affect the document outline and screen readers?
- Would changing a `h2` to `h1` change how the page is navigated?

## Element notes

- `h1`–`h6`: heading elements with opening/closing tags. Use one `h1` per page and logical nested headings for structure.
